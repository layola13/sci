# SA 卫生宏实现计划（最终版）

  ## Summary

  在 src/flattener.zig 中实现卫生宏。核心改动三处：(1) 新增 token 级替换函数替代 std.mem.replaceOwned，(2) 只收集宏体内定义的名称做卫生化，(3) 展开
  顺序改为先卫生后参数。

  ## 文件变更：src/flattener.zig

  ### 1. 新增 renderWithTokenReplacements（~line 259 附近，紧接 renderWithReplacements）

  fn renderWithTokenReplacements(
      allocator: std.mem.Allocator,
      text: []const u8,
      replacements: []const Replacement,
  ) ![]const u8

  逐字符扫描 text，按以下规则处理：

  - " 字符串：原样输出直到闭合 "
  - // 注释：原样输出到行尾
  - @/%/# 前缀：原样输出前缀 + 后续完整标识符（不替换）
  - 标识符 [a-zA-Z_][a-zA-Z0-9_]*：收集完整标识符后在 replacements 中做精确匹配（needle 必须等于整个标识符），匹配则输出 replacement，否则原样输出
  - 其他字符：原样输出

  复用 line_classifier.zig 中已有的 isIdentChar（或在 flattener.zig 中内联同样逻辑）。

  ### 2. 新增 collectDefinedNames

  fn collectDefinedNames(
      allocator: std.mem.Allocator,
      lines: []const SourceLine,
      start: usize,
      end: usize,
  ) !std.StringHashMap(void)

  遍历 lines[start..end]，只收集定义的名称：

  - .label 行：parts[0] 以 L_ 开头 → 加入集合
  - .instruction 行且 inst_form 为以下之一：alloc, stack_alloc, load, borrow, move_, assign, op, ptr_add, call, call_indirect, try_, trunc, zext,
    sext, bitcast, cmpxchg, atomic_load, atomic_rmw, fence, raw_cast, assume_safe, assume_borrow → 如果 parts[0] 以 _ 开头，加入集合

  不收集：.release（!name）、.jmp、.br、.br_null、.return_、.store、.panic 的操作数，以及 %param、@func、#def、字符串、注释。

  ### 3. 新增 buildHygieneReplacements

  fn buildHygieneReplacements(
      allocator: std.mem.Allocator,
      defined_names: *std.StringHashMap(void),
      expansion_id: u64,
  ) !std.ArrayList(Replacement)

  对 defined_names 中每个名称 name，生成：

  - needle = name
  - replacement = name ++ "__sa_hyg" ++ id

  使用 std.fmt.allocPrint 构建 replacement 文本。

  ## 命名约定（写入文档）

  - 卫生后缀：__sa_hyg<N>，N 为全局展开计数器
  - 宏体内 L_ 标签默认私有；如需导出，通过 %label 参数传入
  - __sa_hyg 前缀为保留前缀，用户不得手写
