# 09. Airlock: SA 与 UI 的桥梁

在 SAX 架构中，逻辑层（SA-ASM）运行在隔离的计算核心或 WASM 沙箱中，而视图层运行在 UI 渲染器中。**Airlock (气闸舱)** 是它们之间唯一的通信通道。

## 1. 物理隔离
- **Logic Side (Left)**：拥有线性所有权、内存安全保证、极高主频。
- **View Side (Right)**：处理 DOM、事件响应、CSS 布局、动画渲染。

## 2. 状态同步机制
当你修改 `this.count` 时，发生了什么？

1.  **脏标记 (Dirty Marking)**：编译器在生成的 SA-ASM 中插入了代理指令。修改操作会标记该字段为 "Dirty"。
2.  **补丁生成 (Patching)**：在每一帧结束或逻辑循环间隙，SA 运行时扫描脏数据。
3.  **气闸传输 (Flushing)**：仅将**变化的部分**（如 4 字节的整数）打包成二进制补丁，通过物理内存总线或 WebSocket 发往视图层。

## 3. 事件回调循环
当用户点击按钮时：

1.  **事件捕获**：渲染器捕获 `click` 事件。
2.  **消息入舱**：渲染器将事件名称和参数序列化，推入 Airlock 的入站队列。
3.  **逻辑触发**：SA 逻辑层从队列中取出消息，映射到对应的 `@increment` 函数执行。

## 4. 性能优势：消灭 VDOM
React 等框架使用 VDOM 比较来确定更新，这在复杂应用中非常耗时。
Airlock 架构下，逻辑层直接知道**哪一个内存地址**发生了变化，因此它直接指示渲染器："更新 ID 为 101 的文本节点的 content"。这种**精准打击**消灭了大规模的树遍历。

## 5. Airlock 安全性
气闸舱协议是强类型的。如果视图层尝试向逻辑层发送一个未定义的字段，Airlock 协议层会直接拦截并抛出异常，防止内存污染。
