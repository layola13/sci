SA Whitepaper v0.1

SA (Symbolic Affine) is a fully independent, line-oriented affine ownership language
designed for the LLM era. It produces native executables, WebAssembly modules, and
linkable object files without depending on any host language runtime.

Identity:
- Not an IR: SA is a standalone language with its own CLI, interpreter, and system primitives.
- Not host-dependent: sa run hello.sa executes directly. No Go/Rust/Zig/Python needed.
- Full-platform: Native (x86_64/ARM64/Win/Linux/macOS) + WASM (wasm32/wasm64) + .o for any C-ABI project.

What ships today (v0.1 baseline):
- sa run <file>       : Flattener -> Referee -> Interpreter -> stdout + exit code
- sa build-exe <file> : Flattener -> Referee -> LLVM-C bitcode builder -> .sa.bc -> zig cc -> standalone .exe
- sa build-wasm <file>: Flattener -> Referee -> LLVM-C bitcode builder -> .sa.bc -> zig cc -target wasm32-wasi -> .wasm
- sa build-obj <file> : Flattener -> Referee -> LLVM-C bitcode builder -> .sa.bc -> zig cc -c -> .o (C-ABI linkable)

No Zig source generation. No AST construction. No hidden round-trips.

Checked-in sa_std archive:
- SA-facing std modules (io, fs, net, fmt, process) are assembled with @import on the SA side and @extern declarations in their .sai files.
- C/C++ host examples still use #include, but only in host code.
- The repository checks in artifacts/sa_std/libsa_std.a.
- Rebuild it with zig build sa-std-static -Doptimize=Debug.
- The implementation source is src/runtime/sa_std.zig; the public C header is src/runtime/sa_std.h.

Core Symbols (5 ownership operators + 1 escape):
=       allocate / bind                    target -> Active
&       borrow (read/write by Referee)     source -> Locked_Read/Locked_Mut, borrow -> Active(BorrowView)
^       move / consume                     source -> Consumed
!       release (borrow->unlock; own->free) target -> Consumed
*       raw pointer escape (ffi_wrapper)   produces Untracked register
$...$   native code escape block           conservatively consumes referenced registers

No &mut in syntax. Shared vs exclusive decided by Referee bitmask dynamically.

Instruction Set (ISA):

Memory:
  reg = alloc N
  reg = stack_alloc N                (lifetime bound to function, no ^ escape)
  dst = load src+offset [as T]
  store dst+offset, val [as T]
  dst = take src+offset
  dst = ptr_add base, offset         (derives InteriorPtr)

Arithmetic & Logic:
  Integer: add sub mul sdiv udiv srem urem neg
  Bitwise: and or xor shl lshr ashr not
  Compare: eq ne slt sle sgt sge ult ule ugt uge
  Float:   fadd fsub fmul fdiv fneg
  Float cmp: fcmp_eq fcmp_ne fcmp_lt fcmp_le fcmp_gt fcmp_ge
  Convert: trunc zext sext fptosi sitofp uitofp fptrunc fpext bitcast
  SIMD:    add.v128 sub.v128 mul.v128 shuffle.v128 extract_lane insert_lane

Atomics:
  dst = atomic_load src+offset [ordering]
  atomic_store dst+offset, val [ordering]
  dst, ok = cmpxchg target+offset, expected, new [success_ord] [failure_ord]
  dst = atomic_rmw_<OP> target+offset, val [ordering]
    OP: add sub and or xor xchg smin smax umin umax
  fence [ordering]
  Ordering: relaxed acquire release acq_rel seq_cst

Control Flow:
  jmp L_NAME
  br cond -> L_TRUE, L_FALSE
  br_null reg -> L_NULL, L_NOT_NULL
  call @func(args) / call_indirect func_ptr(args)
  return [reg]

Error Propagation:
  v = ? res              (early return if status != 0)
  panic(code)            (unrecoverable termination)
  panic_msg(code, *s, len)
  Function suffix !      (@f() -> i32! = fallible ABI {u32 status, i32 value})

FFI Airlock (only inside @ffi_wrapper):
  raw = *safe
  safe = assume_safe raw
  view = assume_borrow raw [, mut]

Capability Mask (10-bit, u16):
  0x0001 Active
  0x0002 Locked_Read
  0x0004 Locked_Mut
  0x0008 Consumed
  0x0010 BorrowView
  0x0020 FfiBorrow
  0x0040 Untracked
  0x0080 Fallible
  0x0100 Immutable
  0x0200 InteriorPtr

Function Signature Rules:
  by-value:  @f(x: i32)         native numeric types only
  borrow:    @f(r: &ptr)        ty must be ptr
  move:      @f(^d: ptr)        ty must be ptr
  fallible:  @f() -> i32!       returns {status, value}
  ffi raw:   @extern f(*p: ptr) only in @extern/@ffi_wrapper
  No user-defined type names in signatures.

Preprocessor:
  #def NAME = VALUE              text substitution + constant folding
  #loc "file":line:col           upstream source mapping
  [MACRO] NAME %p1 ... [END_MACRO]
  [REP N] ... [END_REP]          compile-time unrolling with %i
  EXPAND NAME arg1, arg2
  @const NAME = <literal>        global rodata, no type annotation

System Primitives (@sys_*):
  @sys_print(*msg, len)                    Native: write(1,...) / WASM: fd_write
  @sys_read_file(*path, plen, *out_len)    Native: open+read+close / WASM: path_open+fd_read
  @sys_write_file(*path, plen, *data, dlen) Native: open+write+close / WASM: path_open+fd_write
  @sys_exit(code)                          Native: _exit / WASM: proc_exit
  @sys_argv(i) / @sys_argc()               Native: process args / WASM: args_get

FFI Airlock Rules:
  - @ffi_wrapper is the ONLY place for * / assume_safe / assume_borrow
  - Ordinary @func using these -> Trap: IllegalUnsafeContext
  - FFI memory enters via assume_borrow only (no ownership transfer in)
  - Handle/ID pattern for long-lived host objects
  - @extern declares C-ABI symbols; @export exposes without name mangling

Control Flow and Phi:
  - All blocks end with jmp/br/br_null/return.
  - Phi consistency is a frontend responsibility.
  - libsa_scope is the optional helper for explicit !reg cleanup text.

Build Modes:
  - sa build-exe and build-wasm default to --release; no Referee runtime ships in release artifacts.
  - -g enables DWARF and upstream source mapping.
  - --no-debug strips debug metadata.
  - --debug-gas inserts gas counters and may trap with GasExceeded.
  - --debug-san inserts runtime alloc/free bookkeeping for UAF / Double-Free and is slower.

Frontend Contract (R20):
  - Frontends own scope exit: emit explicit !reg for every still-live register.
  - Frontends own Phi consistency: repair mask disagreements before emission.
  - Frontends own lowering: match, async, and hidden Drop insertion are upstream responsibilities.
  - libsa_scope is the optional helper for these repairs.

Design Principles:
  1. Zero AST: flat arrays + u32 indices
  2. Linear scan: single-pass forward, no backtracking
  3. O(1) bitmask: ownership = AND/OR on u16
  4. Five-symbol contract: = & ^ ! *
  5. Frontend responsibility: Drop/Phi/monomorphization are upstream's job
  6. Upstream traceable: #loc -> DWARF -> gdb/lldb
  7. Explicit over implicit: no GC, no hidden Drop, no exceptions
  8. Airlock isolation: unsafe confined to @ffi_wrapper
  9. Full independence: SA runs standalone, interop is optional

Interoperability:
  - @export: unmangled C-ABI symbols callable from C/C++/Rust/Go/Zig
  - @extern: external C-ABI symbols from any library
  - sa build-obj: .o linkable with zig cc / gcc / clang / cargo
  - FFI safety: external pointers enter via assume_borrow only

Testing Model:
  - sa run test.sa + exit code for pass/fail
  - ASSERT_EQ / ASSERT_TRUE macros in sa_core.sa
  - Must-trap tests for Referee validation
  - @export + external test frameworks
  - 32 property-based tests x 100+ iterations

Rust -> SA -> LLVM Bitcode Examples:
  - Struct field access: v.x + v.y -> #def Vec3_x/+0, Vec3_y/+4, load v+Vec3_x -> gep + load
  - Option + ?: let x = read()?; -> res = call @read(...); x = ? res -> extractvalue + icmp + br
  - dyn Trait + VTable: obj.draw() -> @const VT = vtable { draw = @draw }; call_indirect -> load ptr + indirect call
  - async single poll: fetch().await? -> ctx state labels + poll fn + pending path -> label dispatch + branches
  - Rc clone + drop: a.clone(); drop(b) -> atomic_rmw_add / atomic_rmw_sub on Rc_strong -> atomicrmw add/sub

Pilot Protocol:
  - Generate 30 zero-shot prompts: 10 base use cases x 3 variants each.
  - Run against GPT-4o, Claude Opus, and DeepSeek-Coder.
  - Record the first-pass Referee pass rate as the observed baseline.
  - If the baseline drops below 50%, revisit whether to add a text-level pseudo-nested frontend before MVP freeze.
